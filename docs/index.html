What ObsidianX actually is

It's not a wrapper around a vector database. It's not a cloud-managed memory service. It's a three-process system that runs entirely on your machine and integrates Claude Code as a peer, not a guest.

🧠

Markdown vault as ground truth

Your .md files are the database. Edit them by hand, in Obsidian, in VS Code — Claude reads the same source. No re-indexing pipeline.

🛰️

MCP server for Claude Code

Features, with context

What's behind each surface — and what was rebuilt this week.

3D physics graph that reacts to you

Every node is a Markdown file. Every edge is a wiki-link or a semantic spring computed from embeddings. The Real Brain camera mode follows the freshest activity — a save, a delete, a click, an MCP read — and weights them by rarity and recency.

  • Birth / edit / death halos — animated lifecycle states sit in the same code path as physics
  • Translucent cluster bubbles with category-colored equator rings — clicks pass through to nodes inside
  • Octree spatial index — Barnes-Hut force calculation, smooth at hundreds of nodes
  • 2D fast renderer — DrawingContext-based, retained-mode, identical interaction model

2D mode for thousand-node brains

One toggle swaps the 3D Viewport for a DrawingContext-based renderer that handles thousands of nodes without dropping below 60 fps. Same data source, same hit-test, same camera animations — just a different draw call.

  • Categorical coloring — every node tinted by its KnowledgeCategory so regions read at a glance
  • Pan + zoom + click with the same gestures as 3D
  • Hover-scoped wheel zoom — wheel only zooms when cursor's over the map; dashboard scroll is unaffected

Knowledge growth as a real time-series

Not a snapshot — a cumulative chart over actual CreatedAt dates with per-category sub-lines. The vertical staircase exposes burst-learning days vs slow weeks, and the locale-aware date axis means Thai dates render correctly without manual formatting.

  • Honest expertise scoring — relative-to-max rank, not a clamped min(1, sum/10) that pinned every category to 100%
  • Stable through re-indexes — IDs are SHA-256 of normalized path, so growth is monotonic across runs

AI Brain Assistant — pick a backend, ask anything

Local Ollama or one of 130+ cloud models (NVIDIA NIM free tier: Llama 4 / GPT-OSS / DeepSeek / Qwen / Gemma; OpenRouter; DeepSeek). The vault becomes the system prompt. An optional traffic redirect quietly forwards Claude Desktop calls to a local model so the private brain never leaves the machine.

  • One-click model swap across providers
  • Per-router stats — bytes in / out / request count visible
  • Streaming output — tokens render as they arrive

Install — one command, four ways

All four install surfaces hit the same shared installer in ObsidianX.Core.Services.ClaudeBrainRulesInstaller, so memory rules, MCP registration, Ollama setup, and embedding precompute all use one code path. No drift.

Option A — CLI (recommended)

obsidianx-mcp install --vault G:\Obsidian --pull-models --precompute

Single command does: seed/upgrade memory rules → print MCP registration command → probe Ollama → pull nomic-embed-text + gemma3:4b if --pull-models → run embedding precompute if --precompute.

Option B — npm wrapper

npm install -g obsidianx-mcp
obsidianx-mcp install --vault ~/Obsidian

The npm package is a thin Node.js wrapper that locates a co-shipped binary, falls back to a system-installed obsidianx-mcp, and finally falls back to dotnet <dll> if a build is available. postinstall tells you which path it took.

Option C — Desktop UI

Launch ObsidianX.Client.exe once. The constructor calls ClaudeBrainRulesInstaller.EnsureInstalled automatically, seeding the memory directory. The Settings view has dedicated buttons for "Install in Claude Code CLI" and "Install hook" if you want to re-run them.

Option D — automatic on every MCP boot

obsidianx-mcp.exe calls EnsureInstalled in Program.Main before the JSON-RPC loop. Every Claude Code session that spawns the MCP re-checks rule versions and upgrades transparently. The on-disk version-field guard prevents clobbering hand-edited rule files.

Brain-first protocol — enforced from install onward

Every Claude session that touches this vault is wrapped in four layers of policy. They compound: when one is bypassed, the next catches it.

📜

1 — MCP Initialize instructions

tools/list and initialize responses ship a structured imperative protocol: "BEFORE answering, run brain_search; AFTER >2 tool calls, save; AT session end, write a #session-handoff note." Claude sees this on every MCP handshake.

🗂️

2 — Three project memory rules

Auto-installed at ~/.claude/projects/<vault-slug>/memory/:

  • feedback_brain_proactive_save.md
  • feedback_consult_brain_proactively.md
  • feedback_session_handoff_pattern.md

Each rule carries a version: field. Files without one are treated as user-customized and never overwritten.

⚡

3 — UserPromptSubmit hook (gate)

Before Claude sees the user's prompt, the gate injects a brain-first reminder. If the previous turn made code changes without consulting the brain, the gate escalates to a STRONGER NUDGE with a citation requirement.

🛑

4 — Stop hook (audit)

When Claude stops, the hook scans the turn's tool calls. If 3+ edits happened without a single brain_create_note, brain_append_note, or brain_remember, it injects a stronger nudge demanding a save before the session ends.

Result — measurable, not vibes

A diagnostic script (~/.claude/scripts/brain-stats.ps1 -Compact) surfaces token cost vs savings every SessionStart. The line flips from NET COST to NET SAVE once the brain is populated and the hooks are firing — and a recent fix uncovered a logger bug that had been silently reporting "0% hit rate" while every search was actually returning real hits.
